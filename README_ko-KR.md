## Muse Image란?

**Muse Image**는 Meta가 2026년 7월 공개한 에이전트형 이미지 생성 및 편집 모델입니다. 검색과 코드 도구를 사용하고, 추론 시 추가 계산으로 결과를 점검하고 개선하며, 여러 참조 이미지를 결합하고 장면의 나머지를 보존한 채 국소 편집을 수행할 수 있습니다. Meta AI와 일부 소비자 제품에 순차 적용 중이며 이용 범위는 제품과 지역에 따라 다릅니다.

이 모음은 공식 기능 시연과 커뮤니티 프롬프트 사례를 구분하고, 각 사례에 공개 출처, 제작자, 결과 미디어, 프롬프트를 추출하거나 정리한 방식을 남깁니다.

- **네이티브 멀티모달 입력** - 하나의 요청에서 텍스트 지시와 한 장 이상의 참조 이미지를 함께 사용할 수 있습니다
- **생성과 편집** - 새 이미지 생성, 참조 이미지 재스타일링, 자연어 기반의 특정 시각 수정이 가능합니다
- **모델이 결과를 스스로 검토하게 하기** - 에이전트 도구 사용과 자기 개선으로 사실성, 레이아웃, 세부 표현을 높입니다
- **대화형 개선** - 전체 브리프를 다시 쓰지 않고 구도, 피사체, 조명, 스타일을 계속 조정할 수 있습니다
- **전체 프레임을 다시 만들지 않고 편집** - 구도, 정체성, 조명, 영향받지 않는 요소를 유지한 채 대상만 바꿉니다
- **제품별 제공 범위를 확인** - Muse Image는 순차 출시 중이며 이 저장소는 공개 개발자 API를 전제로 하지 않습니다

**조사 출처:** [Meta AI engineering overview](https://ai.meta.com/blog/introducing-muse-image-muse-video-msl/) · [Meta product announcement](https://about.fb.com/news/2026/07/introducing-muse-image-meta-ai/) · [Create images on ImagineVid](https://imaginevid.io/ko/ai-image-generator)

### 프롬프트 변수 재사용

일부 원문에는 `[BRAND]`, `[OBJECT]`, `[NAME]` 같은 대괄호 변수가 있습니다. 값만 바꾸고 검증된 구도, 조명과 재질 구조는 유지하세요.

**예시:**
```
`[BRAND]`를 내 브랜드로, 또는 `[OBJECT]`를 원하는 대상으로 바꾸고 나머지 시각 지시는 그대로 둡니다.
```

변수를 활용하면 출처가 확인된 프롬프트를 처음부터 다시 쓰지 않고 안정적으로 재사용할 수 있습니다.

### No. 2: 장면 흔들림 없는 캐릭터 교체

![Language-EN](https://img.shields.io/badge/Language-EN-blue)
![Featured](https://img.shields.io/badge/Featured-gold-gold)
![Reference](https://img.shields.io/badge/Reference-Image%20Needed-orange)

#### 설명

구도, 의상, 포즈, 가림, 조명, 필름 질감은 그대로 두고 대상 인물만 교체합니다.

#### 원문 프롬프트(영어)

